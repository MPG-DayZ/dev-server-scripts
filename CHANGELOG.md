# Changelog

## 1.4.4

### Добавлено

- Параметр `serverIp` во всех пресетах сервера — настройка IP адреса для подключения клиента (по умолчанию `127.0.0.1`)
- Динамический поиск wildcard-логов в просмотрщике — если файлы не найдены при запуске, logviewer периодически проверяет их появление и автоматически подключает

### Изменено

- Клиент теперь подключается по `serverIp` из пресета вместо захардкоженного `127.0.0.1`
- logviewer при отсутствии файлов больше не завершается с ошибкой — переходит в режим ожидания с повторным поиском

## 1.4.3

### Добавлено

- Подсветка новых уровней логов в просмотрщике: **FATAL** (красный + подчёркивание), **DEBUG** (синий), **TRACE** (тёмно-серый)

### Изменено

- Паттерн `WARNING` теперь также ловит сокращённый вариант `WARN`
- Рефакторинг `Get-ColorForLine` — возвращает структурированный объект с поддержкой ANSI-escape последовательностей вместо строки с именем цвета

## 1.4.2

### Добавлено

- Поддержка diag-режима при очистке логов — определение имени процесса сервера по режиму (`DayZDiag_x64` / `DayZServer_x64_NoBe` / `DayZServer_x64`)
- Защита при загрузке локализации — `try/catch` с fallback на пустые объекты при ошибке чтения `locales.json`
- Защита от null-locale в просмотрщике логов — `trap`-обработчик с fallback на английские сообщения
- Задержка запуска просмотрщика логов вынесена в именованную переменную

### Изменено

- Брендинг: `DayZ Log Viewer` → `MPG Log Viewer` в заголовках окон
- Ренейминг функции: `Normalize-Path` → `ConvertTo-WindowsPath`
- Явное указание UTF-8 при загрузке `locales.json`
- Оптимизация остановки процессов в diag-режиме — клиент не убивается отдельно, т.к. используется один exe

### Исправлено

- Аргументы сервера больше не проходят через преобразование путей, предотвращая искажение строки аргументов
- Добавлена проверка на непустой путь клиентских логов перед поиском файлов
- Заменён wildcard `v*` на точное `v$ScriptVersion` при остановке процессов, исключая случайное убийство процессов других версий

### Технические детали

- Удалён неиспользуемый код системы wildcard-паттернов в просмотрщике логов

## 1.4.1

### Изменено

- Logviewer больше не извлекает версию и путь к клиентским логам самостоятельно — теперь получает их через параметры от `start.ps1`

### Исправлено

- `kill.ps1` теперь точно закрывает окно логвьюера текущей версии вместо маски `v*`

### Технические детали

- Добавлены параметры `-Version` и `-ClientLogsPath` в `scripts/logviewer.ps1`
- Убрано автоопределение пути к клиентским логам через `Get-Process DayZ_x64` в `scripts/logviewer.ps1`
- Убран парсинг `config.ps1` для извлечения версии в `scripts/logviewer.ps1`

## 1.4.0

### Добавлено

- Система версионирования — переменная `$ScriptVersion` в `scripts/config.ps1` как единый источник версии
- Отображение версии в заголовках окон всех скриптов (`start.ps1`, `scripts/kill.ps1`, `scripts/logviewer.ps1`)
- Клавиша `V` в интерактивном меню для показа текущей версии

### Изменено

- Обновлён `scripts/interactive.ps1` для поддержки отображения версии

## 1.3.0

### Добавлено

- **Интерактивный режим выбора пресетов** — удобное консольное меню для выбора пресетов модов и сервера при запуске
  - Навигация стрелками ↑/↓, Home/End, PageUp/PageDown
  - Быстрый выбор по номеру (1-9)
  - Кэширование выбранных пресетов в `.cache/interactive.json`
  - Автоматический выбор при единственном пресете
  - Отмена запуска по Esc или Ctrl+C
  - Включается параметром `"interactive": true` в секции `active` файла `config.json`
  - Параметры командной строки (`-modPreset`, `-serverPreset`) имеют приоритет над интерактивным режимом

### Изменено

- Обновлён `README.md` с документацией по интерактивному режиму
- Добавлен `UPGRADE.md` с инструкцией по добавлению поля `interactive` в существующие конфиги

### Технические детали

- Новый модуль `scripts/interactive.ps1` — переиспользуемый компонент интерактивного меню
- Добавлены строки локализации для интерактивного режима в `scripts/locales.json`
- Кэш хранится в `.cache/interactive.json` с timestamp последнего выбора

## 1.2.0

### Добавлено

- **Logviewer** — встроенный просмотрщик логов в реальном времени (`scripts/logviewer.ps1`)
  - Мониторинг нескольких файлов одновременно (каждые 200 мс)
  - Цветовое разделение серверных/клиентских логов
  - Подсветка ERROR/WARNING/INFO
  - Поддержка пресетов логов (`p_rpt`, `p_adm`, `p_script`, `p_console`, `p_all`)
  - Фильтрация по regex
  - Автоподхват новых файлов и обработка ротации логов
- **Параметры CLI** для `start.ps1` (`-startType`, `-modPreset`, `-serverPreset`, `-autoCloseTime`, `-lang`, `-help`)

### Исправлено

- Исправление некорректной очистки логов
- Фикс невозможности загрузки модов

### Изменено

- QoL улучшения для режима без BattlEye
